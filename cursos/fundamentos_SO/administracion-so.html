Tu primer día como Admin

Usuarios, grupos y el poder de sudo en Linux.

¿Qué significa administrar un SO?

Analogía: Administrar un SO es como ser el gerente de un edificio: decides quién entra, qué llaves tiene cada persona y qué puertas puede abrir.

En Linux, todo gira alrededor de los usuarios. Cada persona que usa el sistema tiene una identidad digital con permisos específicos. El administrador (root) es como el dueño del edificio: tiene acceso total.

Usuarios: Tu identidad en Linux

Usuario Normal

Puede trabajar en su carpeta personal (/home/usuario), ejecutar programas y crear archivos. No puede tocar el sistema.

Root (Superusuario)

Tiene acceso total al sistema. Puede instalar programas, borrar archivos del sistema y cambiar cualquier configuración. ¡Usa con precaución!

El poder de sudo

En lugar de entrar como root (peligroso), Linux nos da sudo: un "pase temporal" para ejecutar UN comando con privilegios de superusuario.

# Instalar un programa (necesita permiso de admin)
sudo apt install htop

# Ver quién soy
whoami

# Convertirme temporalmente en root
sudo su
Regla de oro: Nunca trabajes como root permanentemente. Usa sudo solo cuando sea necesario.

Creando y gestionando usuarios

Como administrador, puedes crear cuentas para otras personas:

# Crear un nuevo usuario
sudo adduser estudiante1

# Ver a qué grupos pertenece
groups estudiante1

# Agregar al usuario a un grupo
sudo usermod -aG sudo estudiante1

# Ver todos los usuarios del sistema
cat /etc/passwd

Grupos: Organizando permisos

Los grupos son como equipos de trabajo. En lugar de dar permisos uno por uno, agrupas usuarios y les das acceso colectivo.

# Crear un grupo nuevo
sudo groupadd ingenieros

# Ver todos los grupos
cat /etc/group

Laboratorio Vivo

Abre tu terminal y practica ser administrador:

# 1. ¿Quién soy?
whoami

# 2. ¿A qué grupos pertenezco?
groups

# 3. Ver la lista de usuarios del sistema
cat /etc/passwd | head -10

# 4. Crear un usuario de prueba
sudo adduser prueba_lab

# 5. Verificar que se creó
id prueba_lab
